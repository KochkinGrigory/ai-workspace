# Создатель Скиллов

Этот скилл помогает создавать хорошо структурированные Agent Skills для Claude Code, следуя лучшим практикам и требованиям валидации.

## Когда использовать этот скилл

Использовать когда:
- Создается новый Agent Skill
- Пишется или обновляется файл SKILL.md
- Проектируется структура скилла и frontmatter
- Решаются проблемы с обнаружением скиллов
- Конвертируются существующие промпты или рабочие процессы в Skills

## Что такое скиллы

Скиллы — это модульные, самодостаточные пакеты, которые расширяют возможности Claude специализированными знаниями, рабочими процессами и инструментами. Это "руководства по вводу в работу" для конкретных доменов — они превращают Claude из универсального агента в специализированного эксперта.

### Что предоставляют скиллы

1. **Специализированные рабочие процессы** — многошаговые процедуры для конкретных задач
2. **Интеграции с инструментами** — инструкции по работе с форматами файлов или API
3. **Доменная экспертиза** — знания компании, схемы БД, бизнес-логика
4. **Исполняемые ресурсы** — скрипты, справочники и ассеты для повторяющихся задач

## Структура скилла

Каждый скилл состоит из обязательного SKILL.md и опциональных ресурсов:

```
skill-name/
├── SKILL.md (обязательно)
│   ├── YAML frontmatter (обязательно)
│   │   ├── name: (обязательно)
│   │   └── description: (обязательно)
│   └── Markdown инструкции (обязательно)
└── Дополнительные ресурсы (опционально)
    ├── scripts/          - Исполняемый код (Python/Bash)
    ├── references/       - Документация для загрузки в контекст
    └── assets/           - Файлы для использования в выводе
```

### SKILL.md (обязательный файл)

**Критичность метаданных:** Поля `name` и `description` в YAML frontmatter определяют, когда Claude использует скилл.

**Язык:** Всё содержимое должно быть на **русском языке**. Технические термины (названия файлов, команд, библиотек) остаются на английском, но описания и инструкции — на русском.

**Стиль написания:** Использовать **императивную форму** (инструкции с глаголом), не второе лицо. Объективный, инструктивный язык:
- ✅ "Чтобы достичь X, выполнить Y"
- ❌ "Вы должны сделать X"

### Принцип Progressive Disclosure

Скиллы используют трехуровневую систему загрузки для эффективного управления контекстом:

1. **Метаданные (name + description)** — всегда в контексте (~100 слов)
2. **Тело SKILL.md** — когда скилл срабатывает (<5k слов)
3. **Дополнительные ресурсы** — по мере необходимости (неограниченно*)

*Скрипты могут выполняться без загрузки в контекст.

### Дополнительные ресурсы (опционально)

#### scripts/ — Исполняемый код

Код (Python/Bash), который выполняет детерминированные операции.

**Когда использовать:**
- Один и тот же код постоянно переписывается
- Требуется надежность и предсказуемость
- Операции можно автоматизировать

**Примеры из других скиллов:**
- PDF: `fill_fillable_fields.py`, `extract_form_field_info.py`
- DOCX: `document.py`, `utilities.py`

**Преимущества:**
- Эффективно по токенам
- Детерминированно
- Может выполняться без загрузки в контекст

#### references/ — Справочные материалы

Документация, загружаемая в контекст по мере необходимости.

**Когда использовать:**
- Для документации, на которую должен ссылаться Claude
- Информация слишком объёмна для SKILL.md
- Содержимое нужно только для конкретных случаев

**Примеры:**
- `finance.md` — финансовые схемы
- `api_docs.md` — спецификации API
- `policies.md` — политики компании

**Лучшая практика:**
- Для больших файлов (>10k слов) включить шаблоны grep поиска
- Избегать дублирования: информация либо в SKILL.md, либо в references
- Хранить только существенные инструкции в SKILL.md

#### assets/ — Файлы для вывода

Файлы, не загружаемые в контекст, а используемые в финальном выводе.

**Когда использовать:**
- Нужны файлы для использования в выводе
- Шаблоны, которые копируются или модифицируются

**Примеры:**
- `logo.png` — брендовые ассеты
- `slides.pptx` — шаблоны PowerPoint
- `frontend-template/` — HTML/React шаблоны

## Инструкции по созданию скилла

### Шаг 1: Определить область действия скилла

Понять, что должен делать скилл:

1. **Задать уточняющие вопросы:**
   - Какую конкретную возможность должен предоставлять скилл?
   - Когда Claude должен использовать этот скилл?
   - Какие инструменты или ресурсы ему нужны?
   - Для личного использования или командного?

2. **Сохранить фокус:** Один скилл = одна возможность
   - ✅ Хорошо: "Заполнение PDF форм", "Анализ данных Excel"
   - ❌ Слишком широко: "Обработка документов", "Инструменты данных"

3. **Собрать конкретные примеры использования:**
   - "Помоги мне повернуть этот PDF"
   - "Создай мне дашборд для отслеживания шагов"
   - Примеры определяют, что скриптовать и что документировать

### Шаг 2: Выбрать расположение скилла

**Личные скиллы** (`~/.claude/skills/`):
- Индивидуальные рабочие процессы
- Экспериментальные скиллы
- Инструменты личной продуктивности

**Проектные скиллы** (`.claude/skills/`):
- Командные рабочие процессы
- Экспертиза, специфичная для проекта
- Общие утилиты (коммитятся в git)

### Шаг 3: Спланировать переиспользуемое содержимое

**Подход "сначала скрипты":** Проанализировать каждый конкретный пример:

1. Рассмотреть, как выполнить пример с нуля
2. Определить, что можно заскриптовать (надежнее генерации текста)
3. Выявить, какие справочные материалы и ассеты будут полезны

**Примеры анализа:**

При создании `pdf-editor` для "Поверни этот PDF":
- Анализ: поворот PDF требует одного и того же кода каждый раз
- Решение: `scripts/rotate_pdf.py` будет полезен

При создании `frontend-webapp-builder` для "Создай приложение задач":
- Анализ: написание фронтенда требует шаблонного HTML/React
- Решение: `assets/hello-world/` с шаблонами будет полезен

При создании `big-query` для "Сколько пользователей вошли сегодня?":
- Анализ: запросы требуют повторного изучения схем
- Решение: `references/schema.md` с документацией схем будет полезен

### Шаг 4: Инициализировать скилл

**Использовать скрипт инициализации:**

```bash
python scripts/init_skill.py <skill-name> --path <output-directory>
```

Скрипт автоматически:
- Создаст директорию скилла
- Сгенерирует шаблон SKILL.md с TODO
- Создаст примеры директорий: scripts/, references/, assets/
- Добавит примеры файлов для настройки

**Примеры:**
```bash
python scripts/init_skill.py my-new-skill --path ~/.claude/skills
python scripts/init_skill.py api-helper --path .claude/skills
```

### Шаг 5: Написать YAML frontmatter

**Требования к полям:**

**name:**
- Только строчные буквы, цифры, дефисы
- Максимум 64 символа
- Должно совпадать с именем директории
- ✅ Хорошо: `pdf-processor`, `git-commit-helper`
- ❌ Плохо: `PDF_Processor`, `Git Commits!`

**description:**
- Максимум 1024 символа
- Включать ЧТО делает И КОГДА использовать
- Использовать конкретные триггерные слова
- Упоминать типы файлов, операции, контекст

**Формула эффективного описания:** `[Что делает] + [Когда использовать] + [Ключевые триггеры]`

**Примеры:**

✅ **Хорошо:**
```yaml
description: Извлечение текста и таблиц из PDF файлов, заполнение форм, слияние документов. Использовать при работе с PDF файлами или когда пользователь упоминает PDFs, формы или извлечение документов.
```

✅ **Хорошо:**
```yaml
description: Анализ электронных таблиц Excel, создание сводных таблиц, генерация графиков. Использовать при работе с Excel файлами, таблицами или анализе табличных данных в формате .xlsx.
```

❌ **Слишком расплывчато:**
```yaml
description: Помогает с документами
description: Для анализа данных
```

**Советы:**
- Включать конкретные расширения (.pdf, .xlsx, .json)
- Упоминать распространённые фразы пользователей ("проанализировать", "извлечь", "сгенерировать")
- Перечислять конкретные операции (не общие глаголы)
- Добавлять контекстные подсказки ("Использовать когда...", "Для...")

**Опциональное поле allowed-tools:**

Ограничить доступ к инструментам (список через запятую):
```yaml
allowed-tools: Read, Grep, Glob
```

Использовать для:
- Read-only скиллов
- Чувствительных к безопасности рабочих процессов
- Операций с ограниченной областью

### Шаг 6: Структурировать содержимое скилла

**Рекомендуемые разделы:**

```markdown
# Название Скилла

Краткий обзор возможностей скилла.

## Быстрый старт

Простой пример для немедленного начала работы.

## Инструкции

Пошаговое руководство для Claude:
1. Первый шаг с чётким действием
2. Второй шаг с ожидаемым результатом
3. Обработка граничных случаев

## Примеры

Конкретные примеры использования с кодом или командами.

## Лучшие практики

- Ключевые соглашения для соблюдения
- Распространённые ошибки, которых следует избегать
- Когда использовать vs. когда не использовать

## Требования

Список зависимостей или предварительных условий:
\`\`\`bash
pip install package-name
\`\`\`

## Продвинутое использование

Для сложных сценариев см. [reference.md](reference.md).
```

### Шаг 7: Реализовать переиспользуемые ресурсы

**Начать с ресурсов:**

1. Создать необходимые скрипты в `scripts/`
2. Подготовить справочные материалы в `references/`
3. Добавить шаблоны и ассеты в `assets/`
4. Удалить ненужные примеры файлов из шаблона

**Примечание:** Может потребоваться ввод от пользователя (например, брендовые ассеты, шаблоны документов).

### Шаг 8: Обновить SKILL.md

Ответить на вопросы:

1. Какова цель скилла в нескольких предложениях?
2. Когда должен использоваться скилл?
3. Как Claude должен использовать скилл на практике?
4. Как использовать каждый включенный ресурс?

**Стиль:**
- Императивная/инфинитивная форма
- Объективный, инструктивный язык
- Технические термины на английском
- Описания на русском

### Шаг 9: Валидировать скилл

**Чек-лист валидации:**

✅ **Структура файлов:**
- [ ] SKILL.md существует в правильном месте
- [ ] Имя директории совпадает с frontmatter `name`

✅ **YAML frontmatter:**
- [ ] Открывающий `---` на строке 1
- [ ] Закрывающий `---` перед контентом
- [ ] Валидный YAML (без табов, правильные отступы)
- [ ] `name` следует правилам именования
- [ ] `description` конкретное и < 1024 символов

✅ **Качество контента:**
- [ ] Чёткие инструкции для Claude
- [ ] Предоставлены конкретные примеры
- [ ] Обработаны граничные случаи
- [ ] Перечислены зависимости (если есть)

✅ **Тестирование:**
- [ ] Описание соответствует вопросам пользователя
- [ ] Скилл активируется на релевантных запросах
- [ ] Инструкции чёткие и действенные

### Шаг 10: Упаковать скилл

**Для распространения использовать скрипт упаковки:**

```bash
python scripts/package_skill.py <path/to/skill-folder>
python scripts/package_skill.py <path/to/skill-folder> ./dist
```

Скрипт автоматически:
1. **Валидирует** скилл, проверяя:
   - Формат YAML frontmatter и обязательные поля
   - Соглашения по именованию
   - Полноту описания
   - Организацию файлов

2. **Упаковывает** скилл в .zip файл, если валидация прошла

Если валидация не прошла, исправить ошибки и повторить упаковку.

### Шаг 11: Тестировать и итерировать

1. **Перезапустить Claude Code** для загрузки скилла

2. **Задать релевантные вопросы:**
   ```
   Помоги мне извлечь текст из этого PDF
   ```

3. **Проверить активацию:** Claude должен автоматически использовать скилл

4. **Проверить поведение:** Убедиться, что Claude следует инструкциям

5. **Итерировать при необходимости:**
   - Замечать трудности или неэффективности
   - Обновлять SKILL.md или ресурсы
   - Тестировать снова

## Диагностика проблем

**Если скилл не активируется:**

1. **Сделать description более конкретным:**
   - Добавить триггерные слова
   - Включить типы файлов
   - Упомянуть распространённые фразы пользователей

2. **Проверить расположение файла:**
   ```bash
   ls ~/.claude/skills/skill-name/SKILL.md
   ls .claude/skills/skill-name/SKILL.md
   ```

3. **Валидировать YAML:**
   ```bash
   cat SKILL.md | head -n 10
   ```

4. **Запустить debug режим:**
   ```bash
   claude --debug
   ```

**Если несколько скиллов конфликтуют:**
- Сделать описания более различимыми
- Использовать разные триггерные слова
- Сузить область каждого скилла

**Если скилл имеет ошибки:**
- Проверить синтаксис YAML (без табов, правильные отступы)
- Проверить пути файлов (использовать прямые слэши)
- Убедиться, что скрипты имеют права на выполнение
- Перечислить все зависимости

## Распространённые паттерны

### Read-only скилл

```yaml
---
name: code-reader
description: Чтение и анализ кода без внесения изменений. Использовать для ревью кода, понимания кодовой базы или документации.
allowed-tools: Read, Grep, Glob
---
```

### Скилл на основе скриптов

```yaml
---
name: data-processor
description: Обработка CSV и JSON файлов с помощью Python скриптов. Использовать при анализе файлов данных или трансформации датасетов.
---

# Data Processor

## Инструкции

1. Использовать скрипт обработки:
\`\`\`bash
python scripts/process.py input.csv --output results.json
\`\`\`

2. Валидировать вывод:
\`\`\`bash
python scripts/validate.py results.json
\`\`\`
```

### Многофайловый скилл с progressive disclosure

```yaml
---
name: api-designer
description: Проектирование REST API следуя лучшим практикам. Использовать при создании API endpoints, проектировании маршрутов или планировании архитектуры API.
---

# API Designer

Быстрый старт: См. [examples.md](examples.md)

Детальный справочник: См. [reference.md](reference.md)

## Инструкции

1. Собрать требования
2. Спроектировать endpoints (см. examples.md)
3. Документировать с помощью OpenAPI spec
4. Проверить по лучшим практикам (см. reference.md)
```

## Лучшие практики для авторов скиллов

1. **Один скилл, одна цель:** Не создавать мега-скиллы
2. **Конкретные описания:** Включать триггерные слова, которые будут говорить пользователи
3. **Чёткие инструкции:** Писать для Claude, не для людей
4. **Конкретные примеры:** Показывать реальный код, не псевдокод
5. **Перечислять зависимости:** Упоминать требуемые пакеты в описании
6. **Тестировать с командой:** Проверять активацию и ясность
7. **Версионировать скиллы:** Документировать изменения
8. **Использовать progressive disclosure:** Размещать продвинутые детали в отдельных файлах

## Ресурсы

Этот скилл включает скрипты автоматизации в директории `scripts/`:

- **init_skill.py** — создание структуры нового скилла из шаблона
- **package_skill.py** — валидация и упаковка скилла в .zip
- **quick_validate.py** — быстрая валидация структуры скилла

См. комментарии в скриптах для деталей использования.

## Выходной формат

При создании скилла:

1. Задать уточняющие вопросы об области и требованиях
2. Предложить имя скилла и расположение
3. Проанализировать примеры для выявления переиспользуемых ресурсов
4. Использовать `init_skill.py` для создания структуры
5. Создать SKILL.md с правильным frontmatter
6. Включить чёткие инструкции и примеры
7. Реализовать необходимые скрипты/references/assets
8. Предоставить инструкции по тестированию
9. Валидировать соответствие всем требованиям

Результат — полный, работающий скилл, следующий всем лучшим практикам и правилам валидации.
